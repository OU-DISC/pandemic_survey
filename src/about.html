About this literature collection

This SurVis browser accompanies a survey of visualization and visual analytics (VIS) work related to the COVID-19 pandemic. It presents the reviewed papers and Survey I–III classifications so readers can explore the literature by keywords, data types, users, visual representations, tasks, journals, and conference/workshop venues.

Abstract

Visualization was pivotal and omnipresent during the COVID-19 pandemic. In particular, interactive dashboards were used widely to communicate vital information to the general public, informing them of the latest status of the crisis. Many dashboards also provide information about the current risks and policies, historical data, predictions, and so on. Based on results from the IEEE VIS Workshop on Visualization for Pandemic and Emergency Responses 2023 (Vis4PandEmRes), and in-depth literature analysis, we systematize the types of visual representations that domain experts have used in different fields as part of their effort to combat the COVID-19 pandemic. Through the lens of these visual representations and a close reading of the relevant literature, we infer the data types used by the domain experts and their visualization tasks. By connecting the dots among data types, visual representations, and visualization tasks, we analyze the strengths and weaknesses of the current capabilities of visualization and visual analytics (VIS) in terms of readiness for future pandemics, identify the needs for improvement, and recommend necessary actions to be taken. Our analysis suggests that advanced VIS techniques could enhance the quality and effectiveness of these works. To address these challenges, the VIS community should make serious efforts to advance VIS as a scientific field and ubiquitous technology. A list of the reviewed papers is included as supplemental material as well as via this SurVis literature browser.

Scope and search themes

Papers were collected around pandemic visualization and related themes, including:

  • pandemic and visualization
  • healthcare and visualization
  • epidemiological modeling and visualization
  • emergency response and visualization

In SurVis these appear as keyword tags such as pandemic, healthcare, epidemiological_modeling, emergency_response, and visualization.

Coding scheme

Surveys I–III used a shared coding scheme to classify papers. The codes below are reflected in the SurVis keyword tags (hover a tag for a short tooltip). Survey II especially deepened codes for data scale, algorithms, interaction, public engagement, and tools/platforms.

Data size (Survey II) — tag prefix size:

Supplement to data-type classification; evaluates the scale of data visualized.

  • [0] size:author_small — author mentioned actual size as small
  • [1] size:author_large — author mentioned size as above small / briefly as big data
  • [2] size:estimated_small — we observed & estimated as small (e.g. one-page table)
  • [3] size:estimated_large — we observed & estimated as above small size

Advanced algorithms (Survey II) — tag prefix advanced_algorithms:

Whether advanced algorithms are employed to support data processing and analysis.

  • [0] advanced_algorithms:no — not using advanced algorithms
  • [1] advanced_algorithms:yes — using advanced algorithms

Level of interaction (Survey II) — tag prefix interaction:

Depth of user–data interaction / visual analytical capability of the system.

  • interaction:basic — Basic
  • interaction:static — Static Vis
  • interaction:medium — Medium
  • interaction:immersive — Immersive
  • interaction:advanced — Advanced

Public engagement (Survey II) — tag prefix engagement:

Whether the visualization is intended for dissemination beyond specialized audiences.

  • [0] engagement:no — no need for dissemination
  • [1] engagement:yes — need for dissemination / public engagement

Tools and platforms (Survey II) — tag prefix tool:

Toolkit/software used and deployment format.

  • tool:toolkits — computational toolkits (e.g. Python, R)
  • tool:web_app — web-based application
  • tool:software — desktop / software tools (e.g. Tableau)
  • tool:mobile_app — mobile application
  • tool:immersive_app — immersive / XR application

Data type — tag prefix data:

Primary input data characterized in the paper, including:

  • data:tabular — tabular data
  • data:timeseries — time-series data
  • data:geospatial — geospatial data
  • data:spatiotemporal — spatiotemporal data
  • data:textual / data:document — text or document corpora
  • data:network — tree, set, graph/network
  • data:event — event data
  • data:imagery — 2D scalar field / imagery
  • data:volume — 3D+ scalar field / volume
  • data:video — video data

User groups — tag prefix user:

Intended or primary users of the visualization, including:

  • user:public, user:policymakers, user:domain_experts
  • user:public_health, user:healthcare_experts, user:epidemiologists
  • user:data_scientists, user:researchers, and other specialist groups when coded

Visual representations — tag prefix visual:

Chart / view types used in the paper (e.g. visual:line, visual:bar, visual:map, visual:dashboard, visual:heatmap, visual:scatter, visual:network, visual:wordcloud).

Visualization levels (Survey I) — tag prefix level:

  • level:disseminative — Disseminative visualization (communicate known facts)
  • level:observational — Observational visualization (explore and confirm patterns)
  • level:analytical — Analytical visualization (interactive analysis)
  • level:model_developmental — Model-developmental visualization (support modeling)

Tasks — tag prefix task:

  • task:visualize — Visualize
  • task:analyze — Analyze
  • task:model — Model
  • task:understand — Understand
  • task:other — Others

Other Survey II flags

  • pandemic_data — uses pandemic-related data
  • domain_specific — domain-specific application (Domain = 1)

How to use this browser

  • Click tags in the keyword, author, journal, or conference/workshop clouds to filter the collection.
  • Combine selectors to refine the result set; sparklines indicate agreement with the current filters.
  • Open a paper entry for metadata, abstract (when available), DOI/URL, and PDF when provided.
  • Use the timeline to explore the literature over years.
  • Hover authorized tags to see coding-scheme definitions.
